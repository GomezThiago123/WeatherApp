EXPLICACION DE LA WEATHER APP

Resumen:
Esta aplicacion muestra una pantalla de clima hecha con React Native y Expo.
La pantalla esta inspirada en un prototipo visual minimalista: fondo blanco,
ciudad centrada, icono grande del clima, metricas secundarias y temperaturas.

Pantalla principal:
El archivo principal es:
app/(tabs)/index.tsx

Esta pantalla muestra:
- Nombre de la ciudad: VILLA LUGANO
- Navegacion entre 3 dias: 1/3, 2/3 y 3/3
- Icono del clima segun la condicion actual
- Humedad
- Presion atmosferica
- Velocidad del viento
- Temperatura minima
- Temperatura actual o promedio
- Temperatura maxima

Datos reales:
La app usa datos reales de la API publica Open-Meteo.
No necesita API key.

La consulta se hace con fetch a una URL de Open-Meteo usando coordenadas de
Villa Lugano / Buenos Aires:
latitude=-34.6751
longitude=-58.4621

La API devuelve datos actuales y diarios, por ejemplo:
- current.temperature_2m
- current.relative_humidity_2m
- current.surface_pressure
- current.wind_speed_10m
- current.weather_code
- daily.temperature_2m_min
- daily.temperature_2m_max
- daily.temperature_2m_mean

Navegacion por dias:
La app guarda un indice en estado usando useState.
Ese indice indica que dia se esta viendo:
- 0 representa 1/3
- 1 representa 2/3
- 2 representa 3/3

Cuando se presiona el boton izquierdo, baja el indice.
Cuando se presiona el boton derecho, sube el indice.
Los botones no permiten salir del rango de dias disponibles.

Estado de carga:
Mientras se espera la respuesta de la API, se muestra:
"Cargando clima..."

Cuando la API responde, se guardan los datos en el estado forecast y se muestra
la pantalla principal.

Icono del clima:
La funcion getWeatherIcon recibe el weatherCode de Open-Meteo.
Segun el codigo, devuelve un icono de @expo/vector-icons:
- weather-sunny
- weather-cloudy
- weather-fog
- weather-rainy
- weather-snowy
- weather-pouring
- weather-lightning

TestID:
La consigna pide testID explicitos para poder evaluar automaticamente la app.
La pantalla incluye estos testID principales:
- screen-weather
- header-city
- button-prev-day
- button-next-day
- navigation-current-day
- icon-weather-*
- metric-item
- metric-icon
- metric-value
- temp-current
- temp-min
- temp-max

Tests:
Los tests estan en:
__tests__/weather-screen.test.tsx

Los tests verifican:
- Que existan los testID obligatorios
- Que se renderice la ciudad
- Que exista el icono climatico
- Que haya 3 metricas secundarias
- Que se muestren temperatura actual, minima y maxima
- Que la navegacion al dia siguiente funcione

Importante:
En los tests no se llama a la API real. Se mockea fetch con datos falsos
controlados para que los tests sean estables y no dependan de internet.

Comandos utiles:
Instalar dependencias:
npm install

Correr la app:
npm start

Correr en navegador:
npm run web

Correr tests:
npm run tests

Verificar TypeScript:
npx tsc --noEmit

Verificar lint:
npm run lint

Archivos importantes:
- app/(tabs)/index.tsx: pantalla principal y logica del clima
- app/(tabs)/_layout.tsx: layout de navegacion sin header visible
- __tests__/weather-screen.test.tsx: tests de cumplimiento
- package.json: scripts, dependencias y configuracion de Jest
- dependencias.txt: instrucciones para instalar y correr el proyecto
