RESUMEN FACIL DE LA WEATHER APP

Que se hizo:
- Se creo una pantalla principal de clima en React Native con Expo.
- La pantalla muestra una ciudad, un icono del clima, temperaturas y metricas.
- Se agrego navegacion entre 3 dias: 1/3, 2/3 y 3/3.
- Se agregaron testID para que la app pueda ser evaluada con tests.
- Se configuraron tests con Jest y Testing Library.
- Se actualizo el proyecto a Expo SDK 55 para que funcione en Expo Go actual.

Archivo principal:
app/(tabs)/index.tsx

Ese archivo hace casi todo lo importante:
- Llama a la API de clima.
- Guarda los datos en estados con useState.
- Cambia de dia cuando se presionan las flechas.
- Decide que icono mostrar segun el codigo del clima.
- Renderiza la pantalla visual.

Datos reales:
La app usa la API publica Open-Meteo.
No necesita usuario, password ni API key.

Se consultan datos de Villa Lugano / Buenos Aires con estas coordenadas:
- latitude: -34.6751
- longitude: -58.4621

Datos que se muestran:
- Temperatura actual
- Temperatura minima
- Temperatura maxima
- Humedad
- Presion atmosferica
- Velocidad del viento
- Icono de condicion climatica

Como funciona la navegacion:
- La app tiene un estado llamado selectedDay.
- selectedDay indica que dia se esta viendo.
- Si selectedDay vale 0, se muestra 1/3.
- Si selectedDay vale 1, se muestra 2/3.
- Si selectedDay vale 2, se muestra 3/3.
- La flecha izquierda resta 1.
- La flecha derecha suma 1.
- No deja pasar por debajo de 0 ni por arriba del ultimo dia.

Estado de carga:
Cuando la app todavia esta esperando datos de internet, muestra:
"Cargando clima..."

Cuando llegan los datos, se muestra la pantalla del clima.

Iconos:
La app recibe un weatherCode de Open-Meteo.
Segun ese numero, se elige un icono:
- Sol
- Nube
- Niebla
- Lluvia
- Nieve
- Tormenta

TestID importantes:
Estos nombres sirven para que los tests encuentren elementos en pantalla:
- screen-weather
- header-city
- button-prev-day
- button-next-day
- navigation-current-day
- icon-weather-*
- metric-item
- metric-icon
- metric-value
- temp-current
- temp-min
- temp-max

Tests:
Archivo:
__tests__/weather-screen.test.tsx

Los tests comprueban:
- Que la pantalla principal exista.
- Que aparezca la ciudad.
- Que haya icono de clima.
- Que haya 3 metricas.
- Que aparezcan temperatura actual, minima y maxima.
- Que al tocar la flecha cambie de dia.

Importante sobre los tests:
Los tests no usan internet.
Se simula la respuesta de la API con datos falsos para que siempre den igual.

Comandos para repasar:
npm install
npm start
npm run web
npm run tests
npm run lint
npx tsc --noEmit

Idea general para explicarlo:
"Hice una pantalla de clima en Expo. La app consume datos reales de Open-Meteo,
los guarda en estado, permite navegar entre tres dias y muestra temperatura,
minima, maxima, humedad, presion y viento. Tambien agregue testID y tests para
cumplir con la consigna de testing."
